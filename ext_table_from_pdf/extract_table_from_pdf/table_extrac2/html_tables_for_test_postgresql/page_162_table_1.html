• To build the PL/Tcl procedural language, you of course need a Tcl installation. The minimum required
NaN version is Tcl 8.4.
• To enable Native Language Support (NLS), that is, the ability to display a program's messages in a
NaN language other than English, you need an implementation of the Gettext API. Some operating systems
NaN have this built-in (e.g., Linux, NetBSD, Solaris), for other systems you can download an add-on package
NaN from https://www.gnu.org/software/gettext/. If you are using the Gettext implementation in the GNU C
NaN library then you will additionally need the GNU Gettext package for some utility programs. For any of
NaN the other implementations you will not need it.
• You need OpenSSL, if you want to support encrypted client connections. OpenSSL is also required
NaN for random number generation on platforms that do not have /dev/urandom (except Windows). The
NaN minimum required version is 1.0.1.
• You need Kerberos, OpenLDAP, and/or PAM, if you want to support authentication using those ser-
NaN vices.
• You need LZ4, if you want to support compression of data with that method; see default_toast_com-
NaN pression and wal_compression.
• You need Zstandard, if you want to support compression of data with that method; see wal_compression.
