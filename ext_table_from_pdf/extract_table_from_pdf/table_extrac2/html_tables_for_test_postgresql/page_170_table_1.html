Note that this setting only affects the main shared memory area. Operating systems such as Linux,
FreeBSD, and Illumos can also use huge pages (also known as “super” pages or “large” pages) auto-
matically for normal memory allocation, without an explicit request from PostgreSQL. On Linux, this
is called “transparent huge pages” (THP). That feature has been known to cause performance degra-
dation with PostgreSQL for some users on some Linux versions, so its use is currently discouraged
(unlike explicit use of huge_pages).
e_page_size (integer)
Controls the size of huge pages, when they are enabled with huge_pages. The default is zero (0).
