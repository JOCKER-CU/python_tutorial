COPY handles this by quoting. A NULL is output as the NULL parameter string and is not quoted, while
a non-NULL value matching the NULL parameter string is quoted. For example, with the default settings,
a NULL is written as an unquoted empty string, while an empty string data value is written with double
quotes (""). Reading values follows similar rules. You can use FORCE_NOT_NULL to prevent NULL
input comparisons for specific columns. You can also use FORCE_NULL to convert quoted null string
data values to NULL.
