Any identity specifications of copied column definitions will be copied. A new sequence is cre-
ated for each identity column of the new table, separate from the sequences associated with the
old table.
LUDING INDEXES
Indexes, PRIMARY KEY, UNIQUE, and EXCLUDE constraints on the original table will be cre-
ated on the new table. Names for the new indexes and constraints are chosen according to the
default rules, regardless of how the originals were named. (This behavior avoids possible dupli-
cate-name failures for the new indexes.)
LUDING STATISTICS
Extended statistics are copied to the new table.
