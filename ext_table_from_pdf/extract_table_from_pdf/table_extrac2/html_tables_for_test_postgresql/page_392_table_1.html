The category and preferred parameters can be used to help control which implicit cast will be
applied in ambiguous situations. Each data type belongs to a category named by a single ASCII character,
and each type is either “preferred” or not within its category. The parser will prefer casting to preferred
types (but only from other types within the same category) when this rule is helpful in resolving overloaded
functions or operators. For more details see Chapter 10. For types that have no implicit casts to or from
any other types, it is sufficient to leave these settings at the defaults. However, for a group of related types
