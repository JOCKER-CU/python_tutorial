According to the SQL standard, the expressions in the output list should be computed before applying
DISTINCT, ORDER BY, or LIMIT. This is obviously necessary when using DISTINCT, since other-
wise it's not clear what values are being made distinct. However, in many cases it is convenient if output
expressions are computed after ORDER BY and LIMIT; particularly if the output list contains any volatile
or expensive functions. With that behavior, the order of function evaluations is more intuitive and there
