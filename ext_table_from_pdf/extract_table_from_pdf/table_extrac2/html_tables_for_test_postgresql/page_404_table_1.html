When dumping logical replication subscriptions, pg_dump will generate CREATE SUBSCRIPTION com-
mands that use the connect = false option, so that restoring the subscription does not make remote
connections for creating a replication slot or for initial table copy. That way, the dump can be restored
without requiring network access to the remote servers. It is then up to the user to reactivate the subscrip-
tions in a suitable way. If the involved hosts have changed, the connection information might have to be
changed. It might also be appropriate to truncate the target tables before initiating a new full table copy.
If users intend to copy initial data during refresh they must create the slot with two_phase = false.
