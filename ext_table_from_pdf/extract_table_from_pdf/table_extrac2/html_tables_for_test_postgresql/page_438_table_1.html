Column Type Description
oid oid\rRow identifier
typname name\rData type name
typnamespace oid (references pg_namespace.oid)\rThe OID of the namespace that contains this type
typowner oid (references pg_authid.oid)\rOwner of the type
typlen int2\rFor a fixed-size type, typlen is the number of bytes in the internal representation of the\rtype. But for a variable-length type, typlen is negative. -1 indicates a “varlena” type (one\rthat has a length word), -2 indicates a null-terminated C string.
typbyval bool\rtypbyval determines whether internal routines pass a value of this type by value or by ref-\rerence. typbyval had better be false if typlen is not 1, 2, or 4 (or 8 on machines where\rDatum is 8 bytes). Variable-length types are always passed by reference. Note that typby-\rval can be false even if the length would allow pass-by-value.
typtype char\rtyptype is b for a base type, c for a composite type (e.g., a table's row type), d for a do-\rmain, e for an enum type, p for a pseudo-type, r for a range type, or m for a multirange type.\rSee also typrelid and typbasetype.
typcategory char\rtypcategory is an arbitrary classification of data types that is used by the parser to deter-\rmine which implicit casts should be “preferred”. See Table 53.65.
typispreferred bool\rTrue if the type is a preferred cast target within its typcategory
typisdefined bool\rTrue if the type is defined, false if this is a placeholder entry for a not-yet-defined type. When\rtypisdefined is false, nothing except the type name, namespace, and OID can be relied\ron.
typdelim char\rCharacter that separates two values of this type when parsing array input. Note that the de-\rlimiter is associated with the array element data type, not the array data type.
typrelid oid (references pg_class.oid)\rIf this is a composite type (see typtype), then this column points to the pg_class entry\rthat defines the corresponding table. (For a free-standing composite type, the pg_class en-\rtry doesn't really represent a table, but it is needed anyway for the type's pg_attribute\rentries to link to.) Zero for non-composite types.
typsubscript regproc (references pg_proc.oid)\rSubscripting handler function's OID, or zero if this type doesn't support subscripting. Types\rthat are “true” array types have typsubscript = array_subscript_handler, but\rother types may have other handler functions to implement specialized subscripting behavior.
