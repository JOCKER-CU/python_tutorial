SQL:2008, SQL:2006, SQL:2003, SQL:1999, and SQL-92. Each version replaces the previous one, so
claims of conformance to earlier versions have no official merit. PostgreSQL development aims for con-
formance with the latest official version of the standard where such conformance does not contradict tra-
ditional features or common sense. Many of the features required by the SQL standard are supported,
though sometimes with slightly differing syntax or function. Further moves towards conformance can be
expected over time.
